### Adding Members to Security Groups

<Steps>
  <Step title="Navigate to Groups in Microsoft Entra Admin Center">
    1. Go to the [Microsoft Entra Admin Center](https://entra.microsoft.com/)
    2. Navigate to **Groups \> All Groups**
    3. Search for "ravical" to find the Ravical security groups
    4. You should see two groups:
       - **ravical-mail-read** (read-only access)
       - **ravical-mail-readwrite** (read and write access)

    ![1 Security Groups Ravical Pn](/images/1.SecurityGroupsRavical.png)
  </Step>
  <Step title="Add Members to ravical-mail-read">
    1. Select the **ravical-mail-read** security group
    2. Click on **Members** in the left sidebar
    3. Click **Add members** at the top of the page
    4. Search for and select the users who need read access to their mailboxes
    5. Click **Select** to add them to the group

    **Who to add:** Most users in your organization who should have their emails accessible by Ravical. You can manually exclude specific users if needed.
  </Step>
  <Step title="New Step">
    1. Select the **ravical-mail-readwrite** security group
    2. Click on **Members** in the left sidebar
    3. Click **Add members** at the top of the page
    4. Search for and select the users who need read and write access
    5. Click **Select** to add them to the group

    **Who to add:** Only designated testers or users who need Ravical to have write access to their mailboxes. This should be a much smaller group than ravical-mail-read.

    ![2 Security Groups Add Member Pn](/images/2.SecurityGroupsAddMember.png)
  </Step>
</Steps>

<Warning>
  **Important Note: **Scoped access through nested groups is not supported by Exchange. You must add users as direct members to these security groups. Adding entire groups as members will not work correctly.
</Warning>
